# Configuration

## Options

```ts
await createBunderstack({
  schema,
  database: { url?: string, authToken?: string, migrations?: string }, // migrations folder, default './migrations'
  access?: Record<string, {
    crud?: boolean,
    ownerColumn?: string,
    list?: 'public' | 'authenticated' | 'owner' | 'deny',
    get?: 'public' | 'authenticated' | 'owner' | 'deny',
    create?: 'public' | 'authenticated' | 'owner' | 'deny',
    update?: 'public' | 'authenticated' | 'owner' | 'deny',
    delete?: 'public' | 'authenticated' | 'owner' | 'deny',
    writableColumns?: string[],
    readonlyColumns?: string[],
  }>,
  auth?: BetterAuthConfig, // native BetterAuth options (except database) ‚Äî see better-auth.com/docs
  storage?: { local: string | true } | { s3: true | { endpoint?: string } },
  storageOptions?: {
    uploadRules?: { allowedMimeTypes?: string[], maxSizeBytes?: number },
    access?: { create?: string, get?: string, delete?: string },
  },
  email?: {
    from: string,
    provider?: 'resend' | 'smtp' | 'console'
      | { send(msg): Promise<{ id?: string }> }, // custom adapter
  },
  env?: {
    server?: Record<string, ZodType>,  // must NOT start with PUBLIC_
    client?: Record<string, ZodType>,  // MUST start with PUBLIC_
    runtimeEnv?: Record<string, unknown>, // browser value source (Vite's import.meta.env)
  },
  trpc?: ((t: BunderstackTRPC) => AnyRouter) | AnyRouter, // builder callback or prebuilt router
  rateLimit?: boolean | RateLimitConfig,
  idempotency?: boolean | IdempotencyConfig,
  realtime?: boolean | { keepaliveMs?: number; bufferSize?: number; redis?: ‚Ä¶ },
})
```

## Database dialect and DATABASE_URL

The dialect comes from your schema (`sqliteTable` ‚Üí SQLite, `pgTable` ‚Üí
Postgres) ‚Äî see [Using PostgreSQL](/docs/getting-started#using-postgresql).
`DATABASE_URL` then selects the engine within that dialect:

| Schema        | `DATABASE_URL`  | Engine                                      |
| ------------- | --------------- | -------------------------------------------- |
| `sqliteTable` | unset           | local SQLite file `./data.db`               |
| `sqliteTable` | `file:./app.db` | local SQLite file                           |
| `sqliteTable` | `libsql://‚Ä¶`    | Turso (`DATABASE_AUTH_TOKEN` for auth)      |
| `pgTable`     | unset           | PGlite in `./data.pglite`                   |
| `pgTable`     | `file:./pgdata` | PGlite in that directory                    |
| `pgTable`     | `memory://`     | PGlite in-memory (tests)                    |
| `pgTable`     | `postgres://‚Ä¶`  | Postgres server (`Bun.sql` under Bun, `postgres`/postgres.js under Node) |

A URL that contradicts the schema's dialect throws a clear error at startup.

## Environment variables

| Variable               | Default                                        | Description                                |
| ---------------------- | ----------------------------------------------- | ------------------------------------------ |
| `DATABASE_URL`         | `file:./data.db` (sqlite) / `file:./data.pglite` (pg) | Connection string ‚Äî see the table above |
| `DATABASE_AUTH_TOKEN`  | ‚Äî                | Turso auth token                           |
| `AUTH_SECRET`          | dev default      | BetterAuth secret ‚Äî required in production |
| `REDIS_URL`            | ‚Äî                | Redis URL for multi-process realtime       |
| `RESEND_API_KEY`       | ‚Äî                | Required when email provider is 'resend'   |
| `SMTP_URL`             | ‚Äî                | Required when email provider is 'smtp'     |
| `S3_BUCKET`            | ‚Äî                | S3 bucket name                             |
| `S3_REGION`            | `us-east-1`      | S3 region                                  |
| `S3_ACCESS_KEY_ID`     | ‚Äî                | S3 access key                              |
| `S3_SECRET_ACCESS_KEY` | ‚Äî                | S3 secret                                  |
| `S3_ENDPOINT`          | ‚Äî                | Custom endpoint for R2 / MinIO             |
