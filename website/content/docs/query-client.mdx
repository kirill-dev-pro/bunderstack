# Query Client

`bunderstack-query` gives you a fully typed client for every CRUD route —
inferred from your server app, with nothing to configure on the client.

## Install

```bash
bun add bunderstack-query @tanstack/react-query
```

## createClient (recommended)

Export the app type from your server entry, then infer everything from it:

```ts
// bunderstack.ts (server)
export const app = createBunderstack({ schema, access, storage })
export type App = typeof app
```

```ts
// api-client.ts
import { createClient } from 'bunderstack-query'
import { QueryClient } from '@tanstack/react-query'
import type { App } from './bunderstack' // type-only import

export const queryClient = new QueryClient({
  defaultOptions: { queries: { staleTime: 30_000 } },
})

export const api = createClient<App>({ queryClient })
```

- **Exposed tables and buckets are inferred** from your access rules and
  storage config — no `tables: [...]` tuples to keep in sync with the server.
  Add a table server-side and `api.newTable` type-checks immediately.
- **Type-only import** — no server code (or Drizzle schema) lands in the
  client bundle.
- **Lazy** — table and bucket clients materialize on first property access.
  One consequence: `Object.keys(api)` is empty by design; access members
  directly.

```ts
api.posts.list({ limit: 20 })
api.posts.get(postId)
api.files.avatars.upload(file) // buckets under api.files.*
```

## Available methods

```ts
// Raw promises
api.posts.list({ limit: 20, offset: 0 })
api.posts.list({ q: 'search term' }) // if searchableColumns set
api.posts.list({ userId: id }) // filterableColumns → WHERE userId = ?
api.posts.list({ id: [a, b, c] }) // array value → WHERE id IN (a, b, c)
api.posts.get(postId)
api.posts.create(data)
api.posts.update(postId, patch)
api.posts.delete(postId)

// TanStack Query option factories
api.posts.listQuery({ limit: 20, offset: 0 })
api.posts.listInfiniteQuery({ limit: 20 }) // keyset cursor pagination
api.posts.getQuery(postId)
api.posts.createMutation()
api.posts.updateMutation()
api.posts.deleteMutation()
```

The server caps every list request at `MAX_LIST_LIMIT` (200), exported from
both `bunderstack` and `bunderstack-query`.

## In a component

```tsx
import { useQuery, useMutation } from '@tanstack/react-query'
import { api } from '~/api-client'

function PostList() {
  const { data } = useQuery(api.posts.listQuery({ limit: 20, offset: 0 }))
  // data: { items: Post[], limit: number, offset: number }

  const create = useMutation(api.posts.createMutation())

  return (
    <ul>
      {data?.items.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

## Prefetching in loaders

```ts
// TanStack Start route loader
export const Route = createFileRoute('/')({
  loader: async () => {
    return queryClient.ensureQueryData(
      api.posts.listQuery({ limit: 20, offset: 0 }),
    )
  },
})
```

## Query invalidation

Pass `queryClient` when creating the client and mutations automatically
invalidate the affected table's queries on success.

## trpc — `api.trpc`

When your server declares a tRPC router, `api.trpc` is a fully typed
`@trpc/tanstack-react-query` proxy — no codegen needed:

```ts
import { createClient } from 'bunderstack-query'
import type { App } from './bunderstack'

const api = createClient<App>({ queryClient })

// Typed option factories — wrap with useQuery / useMutation
const { data } = useQuery(api.trpc.hello.queryOptions())
const { data: me } = useQuery(api.trpc.me.queryOptions())
const greet = useMutation(api.trpc.greet.mutationOptions())
```

Requires `@trpc/client`, `@trpc/tanstack-react-query`, `@trpc/server`, and
`superjson` as peer dependencies. See [tRPC Endpoints](/docs/trpc) for the
server-side setup.

## Explicit alternatives

When you want explicit control instead of app-type inference, the builder API
still works:

```ts
import { createBunderstackQueryClient } from 'bunderstack-query'
import type * as schema from './schema'

// Hand-picked table tuple (type-only schema import)
export const api = createBunderstackQueryClient<typeof schema>().withTables({
  queryClient,
  tables: ['posts', 'user'] as const,
})

// Or schema-as-value (server components, no bundle concern) — exposed tables
// resolved from access rules at runtime
export const api2 = createBunderstackQueryClient().withSchema({
  schema,
  access,
  queryClient,
})
```

## Type utilities

```ts
import type { InferSelect, InferInsert } from 'bunderstack-query'
import type { posts } from './schema'

type Post = InferSelect<typeof posts>
type NewPost = InferInsert<typeof posts>
```
