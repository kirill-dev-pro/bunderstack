# Framework Portability

`app.handler(req: Request): Promise<Response>` — every modern TS framework knows this shape.

## Standalone Bun

```ts
// server.ts
import { app } from './bunderstack'
Bun.serve({ port: 3001, fetch: app.handler })
```

## Next.js (App Router)

```ts
// app/api/[...bunderstack]/route.ts
import { getApp } from '@/bunderstack'

export async function GET(req: Request) {
  return (await getApp()).handler(req)
}
export const POST = GET
export const PATCH = GET
export const DELETE = GET
```

```ts
// bunderstack.ts — lazy singleton for Next.js
import { createBunderstack, type BunderstackApp } from 'bunderstack'
import { provision } from 'bunderstack/provision'
import * as schema from './schema'

let _app: BunderstackApp<typeof schema> | null = null

export async function getApp() {
  if (!_app) {
    _app = await createBunderstack({
      schema,
      auth: { emailAndPassword: { enabled: true } },
    })
    await provision(_app)
  }
  return _app
}
```

Next.js runs on Node, not Bun. If your schema uses `pgTable` and
`DATABASE_URL` points at a real Postgres server, install the Node driver:
`npm install postgres` (postgres.js). Under Bun the built-in `Bun.sql` driver
is used automatically — no extra package needed there.

## TanStack Start: bunderstack-start

The `bunderstack-start` adapter owns all Start-specific glue — the API file
route, SSR-aware fetch, the client setup, session lookup, and the auth client.

```bash
bun add bunderstack-start bunderstack-sync bunderstack-query
```

TanStack Start supports top-level `await`, so `bunderstack.ts` exports `app`
directly, plus the type handle for client inference:

```ts
// src/bunderstack.ts
import { createBunderstack } from 'bunderstack'
import { provision } from 'bunderstack/provision'
import * as schema from './schema'
import { access } from './access'

export const app = await createBunderstack({
  schema,
  access,
  auth: { emailAndPassword: { enabled: true } },
})

export type App = typeof app

await provision(app)
```

Mount the API and set up the whole client:

```ts
// src/routes/api/$.tsx
import { createFileRoute } from '@tanstack/react-router'
import { createApiHandlers } from 'bunderstack-start'
import { app } from '~/bunderstack'

export const Route = createFileRoute('/api/$')({
  server: { handlers: createApiHandlers(app) },
})
```

```ts
// src/api.ts — the entire client setup
import { bunderstackStart } from 'bunderstack-start'
import type { App } from './bunderstack'

export const { createQueryClient, createApi } = bunderstackStart<App>()
```

`createApi(queryClient)` returns the fully inferred
[sync client](/docs/sync-collections) with SSR-aware fetch and
realtime-off-during-SSR already handled.

> **Don't name that file `src/client.ts`.** It's a reserved TanStack Start
> entry-point name — a file called `client.ts` silently replaces the
> framework's hydration entry, and the app renders server-side but is
> completely inert in the browser, with no console error.

Also included:

```ts
import { getSessionUser, createStartAuthClient } from 'bunderstack-start'

// In a server function: resolve the BetterAuth session
const user = await getSessionUser(app, request)

// Browser-side BetterAuth SDK pointed at /api/auth/*
export const authClient = createStartAuthClient()
```
