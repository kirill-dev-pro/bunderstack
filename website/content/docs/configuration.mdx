# Configuration

## Options

```ts
createBunderstack({
  schema,
  database: { url?: string, authToken?: string },
  provision?: boolean | 'auto',
  access?: Record<string, {
    crud?: boolean,
    ownerColumn?: string,
    list?: 'public' | 'authenticated' | 'owner' | 'deny',
    get?: 'public' | 'authenticated' | 'owner' | 'deny',
    create?: 'public' | 'authenticated' | 'owner' | 'deny',
    update?: 'public' | 'authenticated' | 'owner' | 'deny',
    delete?: 'public' | 'authenticated' | 'owner' | 'deny',
    writableColumns?: string[],
    readonlyColumns?: string[],
  }>,
  auth?: BetterAuthConfig, // native BetterAuth options (except database) — see better-auth.com/docs
  storage?: { local: string | true } | { s3: true | { endpoint?: string } },
  storageOptions?: {
    uploadRules?: { allowedMimeTypes?: string[], maxSizeBytes?: number },
    access?: { create?: string, get?: string, delete?: string },
  },
})
```

## Environment variables

| Variable               | Default          | Description                                |
| ---------------------- | ---------------- | ------------------------------------------ |
| `DATABASE_URL`         | `file:./data.db` | libSQL connection string                   |
| `DATABASE_AUTH_TOKEN`  | —                | Turso auth token                           |
| `AUTH_SECRET`          | dev default      | BetterAuth secret — required in production |
| `S3_BUCKET`            | —                | S3 bucket name                             |
| `S3_REGION`            | `us-east-1`      | S3 region                                  |
| `S3_ACCESS_KEY_ID`     | —                | S3 access key                              |
| `S3_SECRET_ACCESS_KEY` | —                | S3 secret                                  |
| `S3_ENDPOINT`          | —                | Custom endpoint for R2 / MinIO             |
