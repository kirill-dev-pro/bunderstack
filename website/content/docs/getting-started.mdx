# Getting Started

## Install

```bash
bun add bunderstack drizzle-orm @libsql/client
bun add -d drizzle-kit
```

## Write your schema

Import Drizzle builders from `bunderstack` to share its Drizzle instance and avoid version conflicts. Re-export Bunderstack's internal tables so migrations include file storage metadata:

```ts
// schema.ts
import { sqliteTable, integer, text } from 'bunderstack'

export * from 'bunderstack/schema'

// BetterAuth required tables — copy full definitions from examples/standalone/schema.ts
export const user = sqliteTable('user', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  email: text('email').notNull().unique(),
  emailVerified: integer('emailVerified', { mode: 'boolean' })
    .notNull()
    .default(false),
  image: text('image'),
  createdAt: integer('createdAt', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updatedAt', { mode: 'timestamp' }).notNull(),
})
// ... session, account, verification (same pattern)

// Your tables
export const posts = sqliteTable('posts', {
  id: integer('id').primaryKey({ autoIncrement: true }),
  title: text('title').notNull(),
  body: text('body').notNull().default(''),
  userId: text('userId')
    .notNull()
    .references(() => user.id, { onDelete: 'cascade' }),
  createdAt: integer('createdAt', { mode: 'timestamp' })
    .notNull()
    .$defaultFn(() => new Date()),
})
```

## Create the app

```ts
// bunderstack.ts
import { createBunderstack } from 'bunderstack'
import * as schema from './schema'

export const app = createBunderstack({
  schema,
  auth: { emailAndPassword: { enabled: true } },
})

if (process.env.NODE_ENV !== 'production') {
  await app.provision()
}
```

`app.provision()` pushes your schema to the database (drizzle-kit push). Call it explicitly in development; omit it in production and use migrations instead.

## Database lifecycle

| Phase | Command | When |
| ----- | ------- | ---- |
| Dev push | `await app.provision()` | Local development |
| Generate | `bunx drizzle-kit generate` | After schema changes |
| Migrate | `bunx drizzle-kit migrate` | Production deploy |

Configure `drizzle.config.ts` with `schema: './src/schema.ts'` and `out: './migrations'`.

## Serve it

```ts
// server.ts
import { app } from './bunderstack'
Bun.serve({ fetch: app.handler })
```

```bash
bun run server.ts
# GET    /api/posts
# POST   /api/posts
# PATCH  /api/posts/:id
# DELETE /api/posts/:id
# POST   /api/auth/sign-up/email
# POST   /api/auth/sign-in/email
# POST   /api/files
# GET    /health
```

The `posts` table has a `userId` column, so Bunderstack auto-wires it: list/get are public, create sets the owner from session, update/delete require the owner.

## Add access rules

To customize per-operation permissions, use `defineAccess`:

```ts
// access.ts
import { defineAccess } from 'bunderstack/access'
import * as schema from './schema'

export const access = defineAccess(schema, {
  posts: {
    ownerColumn: 'userId',
    list: 'public',
    create: 'authenticated',
    update: 'owner',
    delete: 'owner',
    searchableColumns: ['title', 'body'],
  },
})
```

```ts
// bunderstack.ts
export const app = createBunderstack({
  schema,
  access,
  auth: { emailAndPassword: { enabled: true } },
})

if (process.env.NODE_ENV !== 'production') {
  await app.provision()
}
```
